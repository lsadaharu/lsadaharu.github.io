Mysql注入笔记(二)

注入产生的原理

  1. 将用户数据当做信任的数据，没有过滤。
  2. 过滤的规则不严格，数据库配置不恰当。

注入的类型

  1. 按请求方式: GET型，POST型。
  2. 按参数类型：整形，字符型。
  3. 按反馈类型：UNION型，显错型，布尔型，时间型，搜索型等。
  4. insert 注入，update注入，delete注入，二次注入，宽字节注入等。

数据库信息的收集

当判断有注入时，对数据的信息简单收集有助于后续的注入，在显示位输入相关的函数即可。

  • 常用的数据库信息搜集的函数以及方法

    2. 数据库类型: database()
    3. 路径收集:@@datadir
    4. 操作系统:@@version_compile_os
    5. 用户名的收集:系统用户名->system_user(),用户名->user(),当前用户名->current_user(),链接数据库->session_user()

HTTP协议基础

基本概念

  • HTTP协议简称超文本传输协议，用于服务器传输超文本到本地浏览器的传送协议

HTTP协议的特点

  • 支持b/s以及c/s模式
  • 简单快速: 客户向服务器请求服务时，只需传送请求方法和路径。
  • 灵活: HTTP允许传输任意类型的数据对象。
  • 无连接: 无连接的含义是限制每次连接只处理一个请求，服务器处理完客户请求，并收到客户的答应后即断开连接，采用这种方式可以节省传输时间。
  • 无状态: HTTP协议是无状态协议，无状态是指协议对于事务处理没有记忆能力。缺少状态意味着如果后续处理需要前面的信息，则它重新连接，这样可能导致每次连接传送的数据量增加。

URL

  • 定义: 统一资源定位符，是互联网用来标志某一处资源的地址
  • URL的解读:如网址–http://www.helloworld.com:8080/news/index.php?id=1#name
    1. 协议部分为: http://
    2. 域名部分为: www.helloword.com
    3. 端口部分为: 8080
    4. 虚拟目录部分为: news
    5. 文件名部分为: index.php
    6. 参数部分为: id=1
    7. 锚部分为: name

请求的方法

  • GET: 请求指定的页面资源，并返回实体主体。
  • HEAD: 类似get请求，但返回的相应中没有具体的内容，用于获取报文头。
  • POST: 向指定资源提交数据进行处理请求，数据被包含在请求主体中。
  • PUT: 从客户端向服务器端传送的数据取代指定的文档的内容。
  • DELETE: 请求服务器删除指定页面。
  • CONNECT: http/1.1协议中预留给能够将连接改为管道方式的代理服务器。
  • OPTIONS: 允许客户端查看服务端的性能。
  • TRACE: 回复服务器收到的请求，主要用于测试诊断。

Request / Respongse内容信息

  • 请求报文由三部分组成:请求行，消息报头，请求正文。
    • 请求行的内容为请求的方法，请求的URL，HTTP协议版本
    • 请求报文头内容一般含有以下内容
      1. Accept: 显示可以接受的数据类型。
      2. Referer: 请求的来源地址。
      3. Accept-Language: 可接受的文本语言。
      4. User-Agent: 内核，系统，浏览器等信息。
      5. Content-Type: 请求的媒体类型信息也叫MIME。
      6. Host: 请求的主机地址。
      7. Content-Length: 内容的长度
      8. Connection: 对于长链接处理的方式
      9. Cache-Control: 对于缓存的处理
      10. Cookie: 身份认证信息
    • 请求体的内容一般为POST传输的参数值，GET的参数值在请求行中的URL中传输。
  • 响应报文也由三部分组成:状态行，消息报头，响应正文
    • 状态行格式一般为
      HTTP-Version Status-Code Reason-Phrase CRLF
      其中，HTTP-Version表示服务器HTTP协议的版本；Status-Code表示服务器发
      的响应状态代码；Reason-Phrase表示状态代码的文本描述。
      状态代码有三位数字组成，第一个数字定义了响应的类别，且有五种可能取值：
      1xx：指示信息–表示请求已接收，继续处理
      2xx：成功–表示请求已被成功接收、理解、接受
      3xx：重定向–要完成请求必须进行更进一步的操作
      4xx：客户端错误–请求有语法错误或请求无法实现
      5xx：服务器端错误–服务器未能实现合法的请求
      常见状态代码、状态描述、说明：
      200 OK //客户端请求成功
      400 Bad Request //客户端请求有语法错误，不能被服务器所理解
      401 Unauthorized //请求未经授权，这个状态代码必须和WWW-Authenticate报域一起使用
      403 Forbidden //服务器收到请求，但是拒绝提供服务
      404 Not Found //请求资源不存在，eg：输入了错误的URL
      500 Internal Server Error //服务器发生不可预期的错误
      503 Server Unavailable //服务器当前不能处理客户端的请求，一段时间后可能恢复正常
    • 响应报头
      1. location:响应报头域用于重定向接受者到一个新的位置，location常用于在更换域名的时候，
      2. server: 包含服务器用来处理请求的软件信息。
    • 响应实体
      1. Content-Encoding: 实体报头域被用作媒体类型的修饰符。
      2. Content-Language: 实体报头域描述了资源所用的语言。
      3. Content-Length: 实体正文的长度。
      4. Content-Type: 实体报头域发送给接受者的实体正文媒体类型。
      5. Set-Cookie: 服务端设置客户端的Cookie值

渗透windows与linux常用命令

渗透windows常用DOS命令

  1. 查看本机ip信息: ipconfig
  2. 释放ip: ipconfig/release
  3. 重新获取ip: ipconfig/renew
  4. 查看系统信息: systeminfo
  5. 查看局域网计算机列表: arp -a
  6. 查看目录信息: dir
  7. 切换目录: cd 目录名
  8. 打开某个文件: start 文件名
  9. 打开某个网页: start URL
  10. 创建文件内容: echo xxxx > x.txt
  11. 查看文件内容: type xxx.txt
  12. 创建目录: md 目录名
  13. 删除文件夹: rd 文件夹名
  14. 删除文件: del 文件名
  15. 复制文件: copy 文件名
  16. 移动文件或文件夹: move 文件(夹)名 路径
  17. 以树形列出文件夹结构: tree 文件夹名
  18. 查看以启动的服务: net start
  19. 开启或停止某个服务: net start(stop) 服务名
  20. 查看端口占用进程情况: netstat -ano
  21. 查看本机用户信息: net user
  22. 查看用户组信息: net localgroup
  23. 创建用户username: net user username/add
  24. 创建隐藏用户username: net user username$/add
  25. 将用户username添加到管理用户组: `net localgroup administrators username/add
  26. 更改用户username密码: net user username 更改的密码
  27. 删除用户username: net user username/delete
  28. 激活(禁用)用户username: net user username/active:yes(no)
  29. 开启3389: REG ADD HKLM\SYSTEM\CurrentControlSet\Control\Terminal" "Server /v fDenyTSConnections /t REG_DWORD /d 00000000 /f或者wmic RDTOGGLE WHERE ServerName='%COMPUTERNAME%' call SetAllowTSConnections 1
  30. 关闭防火墙: netsh firewall set opmode mode=disable或者netsh advfirewall set allprofiles state off

渗透linux常见命令

  1. 显示主机信息: uname -a
  2. 查看主机名: hostname
  3. 查看ip信息: ipconfig
  4. 查看当前路径: pwd
  5. 查看当前目录文件: ls 查看目录文件详细信息ls -l
  6. 创建目录: mkdir 目录名
  7. 创建新文件: touch 文件名
  8. 为文件创建软连接: ln -s 源文件或者目录 链接文件或者目录
  9. 复制文件或者目录: cp 源文件或目录 目标文件或目录
  10. 删除文件或目录: rm -f强制删除不提醒，rm -r递归删除整个目录树,rm -i删除文件或目录提醒。
  11. 移动文件: mv 源文件或目录
  12. 查找文件: find -name 文件名
  13. 查看文件内容: cat+文件名
  14. 分页显示文件内容: more+文件名
  15. 查看文件属性: file+文件名
  16. gzip压缩打包以及解压:gizp -9+文件，最大压缩比压缩。保留源文件压缩gzip -c 文件名 > 文件名.gz解压gzip -dv+文件名.gz，保留源文件解压gzip -cd 文件名.gz > 压缩文件存放的目录地址
  17. bzip2压缩打包: bzip2+文件名。保留源文件压缩bzip2 -k+文件名.解压bzip2 -dv+文件名.bzip2 .保留源文件解压: bzip2 -kd+文件名.bzip2
  18. tar压缩解压: 以gzip格式压缩tar -zcvf newfile.tar.gz 源文件 gzip格式解压 tar -zxvf 压缩包名 -C 解压的文件放置的目录 以bzip2格式压缩 tar -jcvf newfile.tar.bzip2 源文件 以bzip2格式解压 tar -jxvf 压缩包名 -C 解压的文件放置的目录 查看压缩包中的内容tar -tvf 压缩包名
  19. 改变文件权限: chmod (r->4 w->2 x->1)777 文件名或者按user(所有者)->u，group(组)->u，others(其他人)->u,三套属性a.r->(可读),w->(可写),x->(可执行)赋予权限，如chmod u+r test.c->赋予test.c所有者可读权限.chmod a+w test.c->赋予test.c所有者，组，以及其他人可写权限。
  20. 更改文件/目录所属的拥有者和组: chown 拥有者名:组名 文件或目录名 只修改所属组: chogrp -R 组名 文件或目录名
  21. 添加用户: adduser 用户名 修改密码 passwd 用户名删除用户 userdel 用户名
  22. 查看目前登陆身份: whoami
  23. 查看目前登陆的用户和组: id
  24. 显示磁盘可读的情况: df -h
  25. 显示最后登陆的用户: last
  26. 查看进程: ps -aux 进程的动态情况 top
  27. 终止进程: kill -9 PID
  28. 显示进程ID的网络接口: netstat -tulpn
  29. 启动ip转发: echo '1' > /proc/sys/net/ipv4/ip_forward
